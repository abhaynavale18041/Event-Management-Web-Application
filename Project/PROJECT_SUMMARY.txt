HOW TO USE
================================================================================

1. INSTALL XAMPP
   - Download from https://www.apachefriends.org/
   - Install and start Apache and MySQL

2. COPY TO XAMPP
   - Copy 'event_manager' folder to C:\xampp\htdocs\
   - Or use WAMP/LAMP if preferred

3. CREATE DATABASE
   - Open http://localhost/phpmyadmin
   - Import create_db.sql file
   - This creates database with sample data

4. CONFIGURE (if needed)
   - Edit config.php if MySQL password is set
   - Default works with standard XAMPP

5. ACCESS APPLICATION
   - Navigate to: http://localhost/event_manager
   - Or: http://localhost/event_manager/index.php

6. LOGIN AS ADMIN
   - Email: admin@eventmanager.com
   - Password: admin123
   - Or register new account (first user = admin)

================================================================================
DEPLOYMENT NOTES
================================================================================

FOR SHARED HOSTING:

1. Upload all files via FTP/SFTP to public_html or subdirectory
2. Create database in cPanel
3. Update config.php with database credentials
4. Import create_db.sql via phpMyAdmin
5. Set uploads folder permissions to 755 or 777
6. Update SITE_URL in config.php to your domain
7. Test all functionality
8. Consider setting error_reporting(0) in config.php for production

IMPORTANT SETTINGS TO CHANGE FOR PRODUCTION:

In config.php, change:
- error_reporting(E_ALL); → error_reporting(0);
- ini_set('display_errors', 1); → ini_set('display_errors', 0);

Also enable error logging:
- ini_set('log_errors', 1);
- ini_set('error_log', __DIR__ . '/error.log');

================================================================================
SUPPORT AND TROUBLESHOOTING
================================================================================

COMMON ISSUES:

Database connection error
→ Check MySQL is running
→ Verify config.php credentials
→ Check database exists

Image upload not working
→ Check uploads folder permissions (chmod 755)
→ Check PHP upload_max_filesize
→ Check post_max_size in php.ini

Session not working
→ Check session.save_path is writable
→ Verify session extension is enabled
→ Check browser cookies are enabled

CSRF errors
→ Clear browser cache
→ Check session is starting properly
→ Verify CSRF token generation

For more help, see README.txt troubleshooting section.
